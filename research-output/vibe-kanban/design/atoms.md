# Atoms - Vibe Kanban

Atomic elements are the smallest visual building blocks. The descriptions focus on appearance and behavior only.

## Buttons
- Primary button: solid emphasis, used for main actions (create, save, start)
- Secondary button: muted background for secondary actions
- Outline button: minimal, for neutral actions
- Ghost button: text-only or icon-only with hover feedback
- Link button: text action with underline on hover
- Destructive button: red-tinted for irreversible actions
- Icon button: square or compact, houses a single icon

States:
- default, hover (slightly stronger contrast), focus (visible ring), disabled (reduced opacity)
- icon buttons used for toolbars, menus, and inline actions

Sizes:
- xs, sm, default, lg
- icon-only size for toolbar controls

## Icons
- Monoline icon set with small sizes (16-20px) and medium sizes (24px)
- Categories: navigation, actions, status, warnings, files/branches
- Used in buttons, alerts, badges, list items, and headers

## Text styles
- Page title: large, bold
- Section title: medium, semibold
- Card title: medium
- Body: small-to-regular
- Caption/label: small, muted
- Monospace: URLs, logs, IDs, code snippets

## Form inputs
- Text field: single-line input with border, placeholder, focus ring
- Textarea: multiline with fixed min height
- Auto-expanding textarea: grows with content
- Rich text editor: formatted content with inline toolbar
- Select: dropdown with trigger and list
- Switch: pill track with sliding thumb
- Checkbox: small square, checked state with icon
- Toggle group: small segmented control for mode switching
- Search input: icon-leading input with muted background

## Badges, tags, status indicators
- Badge: rounded pill; variants for default, secondary, destructive, and outline
- Status dot: small filled circle to show column state
- Inline status icon: spinner, success, warning, error

## Drag and resize handles
- Card drag surface: entire task card is draggable
- Panel resize handle: thin vertical bar with dotted grip; highlights on hover/focus

## Loading indicators
- Inline spinner for pending states
- Skeleton or muted text placeholders for empty/loading content
