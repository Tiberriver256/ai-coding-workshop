# Pages

Distinct screens aligned to PRD and feature set.

## Projects list
- Purpose: entry point for projects/boards.
- Key elements: create project, project cards, recent activity.

## Kanban board (project)
- Purpose: orchestrate tasks across statuses.
- Key elements: columns, task cards, board search, create task.

## Task detail
- Purpose: view task scope, attempts, and metadata.
- Key elements: overview, attempts list, agent config, audit log.

## Task attempt / agent execution view
- Purpose: monitor live execution and control agents.
- Key elements: log stream, processes list, stop/pause, status timeline.

## Diff review
- Purpose: inspect code changes and provide feedback.
- Key elements: file list, inline/split diff, comments, submit review.

## Approval center
- Purpose: approve or deny tool actions.
- Key elements: permission request cards, rationale input, audit trail.

## Remote session
- Purpose: remote control of local session from web or mobile.
- Key elements: session thread, status, permission prompts, abort control.

## Device pairing (QR)
- Purpose: secure onboarding and device linking.
- Key elements: QR display, fallback link, status, approve/reject UI.

## Devices & connectivity
- Purpose: manage paired devices and connection status.
- Key elements: device list, connection state, re-pair action.

## Integrations
- Purpose: connect GitHub, Azure DevOps, Copilot CLI, Codex CLI.
- Key elements: status badges, connect/disconnect, setup guidance.

## Settings
- Purpose: security, appearance, agents, and system options.
- Key elements: sections for keys, permissions, notifications, density.
